# Contributing

Thanks for your interest in contributing to **cyberdude-app-web-ui**!  
This guide keeps things simple so you can get started quickly.

---


## If you find a bug or want to suggest an improvement:

- Open an issue with the correct prefix in the GitHub repository.
- Add title and description, then create

---


## Fork the Repository
- [Fork](https://github.com/anburocky3/cyberdude-app-web-ui/fork), You can Fork the **cyberdude-app-web-ui** repository to your repository
- After Fork the repository sync your changes in your repository
- Go to your system and Clone your forked repository in your local system
- Go to Terminal 
`npm install`
`npm run dev`

---

## Create a branch

- Go to your terminal

- Check out the branch `git checkout`

- It's in main branch, now create one `git chckout -b "brach-name"`

- Create a branch to work and also to avoid code conflicts from others work

- Ensure you have changed the codes

- Avoid inline styles unless necessary.

---

## Push your changes

- After the code changes, follow these

1. `git add .`
2. `git commit -m "fix:comments"`

### Allowed Types to commit

feat – New feature

fix – Bug fix

docs – Documentation changes

style – Formatting changes (no logic changes)

refactor – Code refactoring

perf – Performance improvements

test – Adding or updating tests

chore – Maintenance tasks

Examples
git commit -m "feat(ui): add responsive navbar"
git commit -m "fix(auth): handle token expiration"
git commit -m "docs(readme): update setup instructions"

3. `git push -u origin branch name or git push git push --set-upstream origin branch name`

- This actions making the code moved to forked
repository.
